In Fide, everyone who participates in work is a **User**. This includes real people, AI agents, and service accounts. **Users can be members of one or more teams in the Fide Workspace.**

<Callout type="info" title="Users vs. Members">
  A **User** is an individual registered with the Fide Workspace. A **Team Member** is a User who has been added to a specific [Team](/docs/workspace/teams).
</Callout>

## Shared Identity

Fide treats all actors—human or agent—with a common foundation. Whether you are an employee or an autonomous agent, you have a unique identity, can own resources, and can be assigned responsibility.

### The Three User Types

| Type | Description | Example |
|------|-------------|---------|
| **Human** | Real people who sign up | You, your teammates |
| **Agent** | AI team members that execute work | Market Scanner, Researcher |
| **Service Account** | Accounts for headless automation | Workflow owners, API bots |

## Participation

A User becomes a productive part of the organization by joining one or more teams. once a user is a **Team Member**, they inherit the trust boundaries and access rights associated with that team.

*   **Identifiable**: Every action in the [Context Graph](/docs/fcp) is linked to a specific User ID.
*   **Accountable**: Decisions made by an agent user are traceable back to the agent's specific version and configuration.
*   **Flexible Access**: Tools like the [Dispatch Engine](/docs/workspace/dispatch) can assign tasks to "The best user for the job," whether that user is a human or an agent.
